# AI芯片测试工作流 - 网站集成配置说明

## 概述

本文档说明如何配置和部署修改后的N8N工作流，使其能够与戴煦个人网站完美集成。

## 工作流结构

### 主要节点

1. **Webhook节点** - 接收网站请求
   - `Webhook - 测试代码生成`: `1e50cbd3-cc31-4ab8-80c4-aa4aadf1e5fc`
   - `Webhook - 测试方案生成`: `f27ab7f0-1c71-4929-8163-24d372a8761b`

2. **工作流路由节点** - 根据请求类型分发到不同的AI Agent

3. **AI Agent节点**
   - `AI Agent`: 负责测试代码生成
   - `AI Agent1`: 负责测试方案生成

4. **响应处理节点**
   - `格式化响应`: 格式化AI输出
   - `Webhook响应`: 返回结果给网站

## 配置步骤

### 1. 部署N8N工作流

1. 将修改后的 `192 (1).json` 文件导入到您的N8N实例中
2. 确保所有Google Docs和DeepSeek API凭据已正确配置
3. 激活工作流

### 2. 获取Webhook URL

工作流激活后，您将获得两个webhook URL：

```
测试代码生成: https://your-n8n-instance.com/webhook/1e50cbd3-cc31-4ab8-80c4-aa4aadf1e5fc
测试方案生成: https://your-n8n-instance.com/webhook/f27ab7f0-1c71-4929-8163-24d372a8761b
```

### 3. 更新网站配置

在 `戴煦个人网站.html` 文件中，找到以下配置部分：

```javascript
const N8N_CONFIG = {
    baseUrl: 'https://your-n8n-instance.com',
    webhookUrls: {
        codeGen: 'https://your-n8n-instance.com/webhook/1e50cbd3-cc31-4ab8-80c4-aa4aadf1e5fc',
        testPlan: 'https://your-n8n-instance.com/webhook/f27ab7f0-1c71-4929-8163-24d372a8761b'
    }
};
```

将 `https://your-n8n-instance.com` 替换为您的实际N8N实例URL。

## 工作流类型

### 1. 测试代码生成 (code/quick)
- **触发条件**: 用户选择"测试代码生成"或"快速测试"
- **处理流程**: Webhook → 工作流路由 → AI Agent → 格式化响应 → Webhook响应
- **输出内容**: 完整的芯片测试代码，包括初始化、CON、FUN、VOH、VOL、VIH、VIL、IIH、IIL、II等测试项

### 2. 测试方案生成 (plan)
- **触发条件**: 用户选择"测试方案生成"
- **处理流程**: Webhook → 工作流路由 → AI Agent1 → 格式化响应 → Webhook响应
- **输出内容**: 详细的测试方案和测试向量表格

## 请求格式

网站发送到N8N的请求格式：

```json
{
    "message": "SN74LS192 测试代码",
    "workflowType": "code",
    "timestamp": "2024-01-01T00:00:00.000Z",
    "source": "website_chat"
}
```

## 响应格式

N8N返回给网站的响应格式：

```json
{
    "success": true,
    "timestamp": "2024-01-01T00:00:00.000Z",
    "workflowType": "code",
    "output": "生成的测试代码内容...",
    "message": "测试代码生成完成",
    "status": "completed"
}
```

## 错误处理

工作流包含完整的错误处理机制：

1. **输入验证**: 检查请求格式和必要字段
2. **路由错误**: 处理无效的工作流类型
3. **AI处理错误**: 捕获AI Agent执行失败
4. **网络错误**: 处理超时和连接问题

错误响应格式：

```json
{
    "success": false,
    "timestamp": "2024-01-01T00:00:00.000Z",
    "workflowType": "code",
    "output": null,
    "message": "工作流执行失败",
    "error": "具体错误信息",
    "status": "error"
}
```

## 故障排除

### 常见问题

1. **Webhook无法访问**
   - 检查N8N实例是否正常运行
   - 验证webhook URL是否正确
   - 确认网络连接正常

2. **AI Agent无响应**
   - 检查DeepSeek API凭据
   - 验证Google Docs访问权限
   - 查看N8N执行日志

3. **响应格式错误**
   - 检查格式化响应节点配置
   - 验证JSON格式是否正确
   - 确认CORS设置

### 调试方法

1. 查看N8N工作流执行历史
2. 检查浏览器开发者工具网络面板
3. 查看N8N和网站的错误日志
